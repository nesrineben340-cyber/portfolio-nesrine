Stage — Pôle Infrastructure SMAG

Administration systèmes & réseaux · Automatisation · Sécurité périmétrique

Stage de 2ème année effectué au sein du pôle Infrastructure de SMAG (Smart Agriculture), leader français des logiciels de gestion agricole. Immersion dans un environnement IT critique supportant des services SaaS utilisés par 30 000 utilisateurs.

Problématique du stage : Comment l'automatisation des processus de surveillance et l'industrialisation de la maintenance permettent-elles de sécuriser une infrastructure supportant des services SaaS critiques ?

🔐 Chapitre I — Ingénierie Réseau & Sécurité Périmétrique

2.1 · Audit et optimisation du pare-feu Fortinet (FortiOS)

  • Analyse des règles de filtrage via les compteurs de trafic (Hit Count) en droits de lecture.
  • Identification des règles obsolètes ou redondantes.
  • Préparation de recommandations : suppression d'objets réseaux inutilisés, regroupement de règles, renommage selon la charte.
  • Application des modifications en binôme avec le tuteur de stage.
Fortinet FortiOS Analyse de règles Sécurité périmétrique

2.2 · Veille et documentation — Sécurisation des accès (802.1X)

  • Étude du standard 802.1X pour la sécurisation des prises réseau physiques (RJ45).
  • Analyse des trois composants : Supplicant (poste client), Authenticator (switch) et serveur RADIUS.
  • Rédaction d'une documentation sur l'isolation des équipements non reconnus (VLAN de quarantaine).
802.1X RADIUS VLANs Documentation technique

2.3 · Analyse comparative des architectures d'hyperviseurs

  • Note documentaire sur les hyperviseurs Type 1 (Bare Metal) : VMware ESXi, Hyper-V — utilisés pour les BDD critiques de SMAG.
  • Comparaison avec les hyperviseurs Type 2 (Hosted) : VirtualBox, VMware Workstation — utilisés pour les tests.
  • Analyse des performances et des cas d'usage dans l'environnement SMAG.
VMware ESXi Hyper-V VirtualBox Virtualisation

🖥️ Chapitre II — Industrialisation & Gestion du Cycle de Vie du Parc

3.1 · Déploiement des postes via PXE (Masterisation)

  • Démarrage des postes sur la carte réseau (PXE) pour récupérer automatiquement une image système.
  • Déploiement d'images Windows préconfigurées (OS + mises à jour + logiciels SMAG).
  • Avantage : homogénéité du parc, gain de temps, conformité garantie dès la mise en service.
PXE Masterisation Windows Server Déploiement réseau

3.2 · Support utilisateur N2 & diagnostic matériel (GLPI)

  • Traitement de tickets via GLPI — prise en main à distance pour diagnostic.
  • Cas concret : panne Wi-Fi/Bluetooth → code erreur 45 (périphérique non détecté) dans le Gestionnaire de périphériques.
  • Diagnostic : carte réseau interne défectueuse (Realtek) → commande et envoi d'une carte de remplacement pour éviter l'immobilisation du poste.
  • Démarche structurée : détection → analyse → résolution → clôture du ticket.
GLPI Prise en main à distance Diagnostic matériel Support N2

3.3 · Gestion du matériel obsolète & conformité RGPD (en binôme)

  • Inventaire et retrait des équipements hors service de la base GLPI.
  • Démontage des disques durs (SSD & HDD) pour destruction physique par prestataire spécialisé — conformité RGPD.
  • Récupération de composants réutilisables (RAM, cartes réseau) pour le stock de maintenance.
GLPI RGPD Décommissionnement Gestion de parc

⚙️ Chapitre III — Automatisation & Supervision Proactive

4.1 · Développement de scripts PowerShell (avec le DSI)

  • Rédaction sous Notepad++ de scripts PowerShell pour l'extraction des journaux Windows (Système, Sécurité, Application).
  • Intégration de la collecte des logs d'erreurs MySQL et SQL Server avec l'appui du DSI.
  • Choix de l'authentification intégrée Windows (Integrated Security=True) pour éviter le stockage de mots de passe en clair.
  • Transmission des logs en temps réel vers un serveur Syslog.
PowerShell Notepad++ SQL Server MySQL Active Directory

4.2 · Ordonnancement & configuration du serveur Syslog

  • Mise en place de tâches planifiées Windows via PowerShell pour automatiser l'exécution des scripts.
  • Configuration du serveur Syslog Linux (nano /etc/rsyslog.conf) : ajout de la plage IP du client, paramétrage UDP/TCP, inclusion de l'IP source.
  • Redémarrage et validation du service : systemctl restart rsyslog.
  • Tests de réception via MobaXterm utilisé comme serveur Syslog de test.
rsyslog Syslog MobaXterm Linux (Ubuntu) Tâches planifiées

4.3 · Sécurisation des accès privilégiés avec KeePass 2

  • Utilisation de KeePass 2 pour la gestion et le stockage sécurisé des mots de passe d'administration.
  • Application des bonnes pratiques de sécurité des accès privilégiés dans un environnement de production.
KeePass 2 Gestion des accès Sécurité
