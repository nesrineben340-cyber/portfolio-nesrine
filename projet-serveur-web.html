Projet — Serveur Web

Déploiement et sécurisation d'un serveur web Apache sous Linux

Projet de déploiement d'un serveur web Apache sous Linux avec configuration des hôtes virtuels, mise en place du protocole HTTPS et sécurisation de l'accès au serveur.

🎯 Objectifs du projet

  • Installer et configurer un serveur Apache2 sous Linux.
  • Mettre en place des hôtes virtuels (Virtual Hosts) pour héberger plusieurs sites.
  • Configurer le protocole HTTPS avec un certificat SSL.
  • Sécuriser le serveur : pare-feu, droits d'accès, désactivation des modules inutiles.
  • Configurer les logs Apache pour la supervision des accès.
  • Tester le bon fonctionnement et la disponibilité du service web.

🏗️ Architecture déployée

🖥️ Serveur Linux (Ubuntu Server) │ ├── 🔥 Pare-feu UFW (ports 80 et 443 ouverts) │ ├── 🌐 Apache2 │ ├── VirtualHost site1.local (port 80 → redirection HTTPS) │ ├── VirtualHost site1.local (port 443 · SSL activé) │ └── VirtualHost site2.local (port 443 · SSL activé) │ ├── 🔒 Certificat SSL (Let's Encrypt / auto-signé) │ └── 📋 Logs Apache ├── /var/log/apache2/access.log └── /var/log/apache2/error.log

⚙️ Étapes de réalisation

1 · Installation du serveur Apache

  • Installation d'Apache2 : apt install apache2.
  • Vérification du statut du service : systemctl status apache2.
  • Activation du démarrage automatique : systemctl enable apache2.
  • Test de la page par défaut Apache depuis un navigateur.
Apache2 systemctl Ubuntu Server

2 · Configuration des hôtes virtuels

  • Création des répertoires de sites dans /var/www/.
  • Rédaction des fichiers de configuration Virtual Host dans /etc/apache2/sites-available/.
  • Activation des sites avec a2ensite et rechargement d'Apache.
  • Modification du fichier /etc/hosts pour la résolution locale des noms.
VirtualHost a2ensite /etc/hosts

3 · Mise en place du HTTPS (SSL/TLS)

  • Activation du module SSL : a2enmod ssl.
  • Génération ou installation d'un certificat SSL.
  • Configuration du VirtualHost en port 443 avec les chemins vers le certificat.
  • Redirection automatique du trafic HTTP (port 80) vers HTTPS (port 443).
  • Vérification du cadenas vert dans le navigateur.
SSL/TLS HTTPS a2enmod ssl Certificat

4 · Sécurisation du serveur

  • Configuration du pare-feu UFW : ouverture des seuls ports nécessaires (80, 443, 22).
  • Attribution correcte des droits sur les fichiers : chown -R www-data:www-data.
  • Désactivation des modules Apache non utilisés pour réduire la surface d'attaque.
  • Masquage de la version d'Apache dans les en-têtes HTTP (directive ServerTokens Prod).
UFW Droits Linux ServerTokens Hardening

5 · Supervision via les logs

  • Consultation des logs d'accès : tail -f /var/log/apache2/access.log.
  • Analyse des logs d'erreur pour détecter les problèmes de configuration.
  • Mise en place d'une rotation des logs avec logrotate.
Logs Apache logrotate Supervision
