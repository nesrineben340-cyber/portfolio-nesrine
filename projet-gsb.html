Projet — GSB

Application web de gestion des frais de déplacement — Galaxy Swiss Bourdin

Projet pédagogique basé sur le contexte fictif de l'entreprise Galaxy Swiss Bourdin (GSB), laboratoire pharmaceutique. L'objectif est de développer une application web permettant aux visiteurs médicaux de saisir leurs frais de déplacement et aux comptables de les valider.

🎯 Contexte & Objectifs

GSB est un laboratoire pharmaceutique fictif dont les visiteurs médicaux se déplacent quotidiennement chez les médecins et pharmaciens. L'ancienne gestion des frais étant papier, le projet vise à numériser et automatiser ce processus via une application web sécurisée avec deux profils : visiteur et comptable.

⚙️ Fonctionnalités développées

Fonctionnalité Profil Statut
Authentification sécurisée (login/mot de passe)Tous✅
Saisie des frais forfaitisés (repas, hébergement, km)Visiteur✅
Saisie des frais hors forfait (justificatifs)Visiteur✅
Consultation de l'historique des fiches de fraisVisiteur✅
Validation / rejet des fiches de fraisComptable✅
Mise en paiement des fiches validéesComptable✅
Tableau de bord comptableComptable✅

🛠️ Technologies utilisées

Back-end — PHP & MySQL

  • Développement en PHP avec architecture MVC (Modèle-Vue-Contrôleur).
  • Base de données MySQL : conception du schéma relationnel (visiteurs, fiches de frais, lignes de frais…).
  • Requêtes SQL préparées pour sécuriser les accès aux données (protection injections SQL).
  • Gestion des sessions PHP pour l'authentification et la gestion des profils.
PHP MySQL Architecture MVC Sessions PHP SQL préparé

Front-end — HTML / CSS

  • Interfaces utilisateur en HTML5 / CSS3.
  • Formulaires de saisie avec validation côté client.
  • Affichage responsive adapté aux différents écrans.
HTML5 CSS3 Formulaires

Environnement de développement

  • Serveur local XAMPP (Apache + MySQL + PHP).
  • Gestion de la base de données via phpMyAdmin.
  • Éditeur de code : Visual Studio Code.
XAMPP phpMyAdmin VS Code Apache
