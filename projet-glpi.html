Projet — Installation GLPI

Déploiement et configuration d'un serveur de gestion de parc informatique

Projet consistant à installer et configurer GLPI (Gestionnaire Libre de Parc Informatique) sur un serveur Linux. GLPI permet de gérer l'inventaire du matériel, les tickets d'assistance et le suivi des actifs informatiques d'une organisation.

🎯 Objectifs du projet

  • Installer un serveur LAMP (Linux, Apache, MySQL, PHP) pour héberger GLPI.
  • Déployer et configurer GLPI avec sa base de données MySQL.
  • Configurer les profils utilisateurs (administrateur, technicien, utilisateur).
  • Mettre en place l'inventaire du parc informatique.
  • Configurer le système de tickets d'assistance (helpdesk).
  • Intégrer le plugin FusionInventory pour l'inventaire automatique des postes.

🏗️ Architecture déployée

🖥️ Serveur Linux (Ubuntu Server) │ ├── 🌐 Apache2 — Serveur web (port 80/443) ├── 🗄️ MySQL — Base de données GLPI ├── 🐘 PHP 8.x — Moteur de traitement │ └── 📦 GLPI (application web) ├── Module Parc → inventaire matériel & logiciels ├── Module Helpdesk → tickets d'assistance ├── Module Contrats → suivi des garanties └── Plugin FusionInventory → collecte automatique

⚙️ Étapes de réalisation

1 · Préparation du serveur Linux

  • Installation d'Ubuntu Server en machine virtuelle.
  • Mise à jour du système : apt update && apt upgrade.
  • Configuration de l'adresse IP statique du serveur.
  • Ouverture des ports nécessaires dans le pare-feu (ufw allow 80).
Ubuntu Server Terminal Linux ufw

2 · Installation de la pile LAMP

  • Installation d'Apache2 : apt install apache2.
  • Installation de MySQL et création de la base de données GLPI.
  • Installation de PHP et des extensions requises par GLPI.
  • Vérification du bon fonctionnement de chaque composant.
Apache2 MySQL PHP 8 LAMP

3 · Déploiement et configuration de GLPI

  • Téléchargement et extraction des sources GLPI dans /var/www/html/glpi.
  • Attribution des droits sur les dossiers : chown -R www-data:www-data.
  • Configuration via l'interface web : connexion à la base MySQL, création du compte admin.
  • Suppression du fichier d'installation pour sécuriser l'accès.
GLPI Configuration web Droits Linux

4 · Configuration des profils et du helpdesk

  • Création des profils : Administrateur, Technicien, Utilisateur final.
  • Configuration des catégories de tickets (matériel, logiciel, réseau…).
  • Paramétrage des règles d'attribution automatique des tickets aux techniciens.
  • Test du cycle complet : ouverture → traitement → clôture d'un ticket.
Helpdesk Profils GLPI Gestion des tickets

5 · Plugin FusionInventory — Inventaire automatique

  • Installation du plugin FusionInventory dans GLPI.
  • Déploiement de l'agent FusionInventory sur les postes clients.
  • Vérification de la remontée automatique des informations matérielles et logicielles.
FusionInventory Agent inventaire Automatisation
